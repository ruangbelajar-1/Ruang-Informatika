1. Struktur Data dan Algoritma Dasar

Apa itu Struktur Data?

Struktur data adalah cara menyimpan dan mengatur data di komputer agar dapat digunakan secara efisien. Bayangkan seperti menyimpan buku di perpustakaan - jika tersusun rapi, mencari buku menjadi lebih mudah!

Analogi: Struktur data seperti lemari pakaian. Jika kaos, celana, dan jaket disimpan terpisah (terstruktur), kita bisa mengambil pakaian lebih cepat dibandingkan jika semuanya dicampur.

Tipe Data Dasar

Tipe DataPenjelasanContoh
IntegerBilangan bulat (tanpa desimal)10, -5, 0, 1000
Float/DoubleBilangan desimal3.14, -0.5, 2.718
StringTeks/kumpulan karakter"Halo", "SMK", "Informatika"
BooleanNilai benar atau salahtrue, false
Array/ListKumpulan data dengan tipe sama[1, 2, 3, 4, 5]

Apa itu Algoritma?

Algoritma adalah langkah-langkah logis dan terurut untuk menyelesaikan suatu masalah. Algoritma harus memiliki sifat:

  • Finite (Terbatas): Memiliki awal dan akhir yang jelas
  • Definite (Tepat): Setiap langkah jelas dan tidak ambigu
  • Input: Memiliki masukan (boleh 0 atau lebih)
  • Output: Menghasilkan keluaran (minimal 1)
  • Effective: Setiap langkah dapat dikerjakan
Contoh Algoritma Sederhana: Membuat Teh Manis

1 Siapkan gelas, teh celup, gula, dan air panas
2 Masukkan teh celup ke dalam gelas
3 Tuangkan air panas ke gelas
4 Tambahkan 2 sendok gula
5 Aduk hingga gula larut
6 Teh manis siap diminum

Komputasi adalah proses menghitung atau memproses data untuk menghasilkan informasi. Komputasi bisa dilakukan secara manual (oleh manusia) maupun komputer (oleh mesin).

3. Pseudocode

Apa itu Pseudocode?

Pseudocode (kode semu) adalah cara menulis algoritma menggunakan bahasa yang mirip bahasa pemrograman tetapi masih mudah dipahami manusia. Pseudocode tidak memiliki aturan sintaks yang ketat seperti bahasa pemrograman sungguhan.

Tujuan Pseudocode:
1. Merencanakan algoritma sebelum coding
2. Memudahkan komunikasi antar programmer
3. Dokumentasi logika program
4. Belajar logika tanpa terikat sintaks bahasa pemrograman

Struktur Dasar Pseudocode

A. Input dan Output
// Membaca input dari pengguna INPUT nama INPUT umur // Menampilkan output ke layar OUTPUT "Halo, " + nama OUTPUT "Umur Anda: " + umur
B. Percabangan (IF-THEN-ELSE)
// Menentukan kelulusan berdasarkan nilai INPUT nilai IF nilai >= 75 THEN OUTPUT "Selamat, Anda LULUS!" ELSE OUTPUT "Maaf, Anda BELUM LULUS." ENDIF
C. Perulangan (FOR dan WHILE)
// Perulangan FOR: Cetak angka 1 sampai 5 FOR i FROM 1 TO 5 DO OUTPUT i ENDFOR // Perulangan WHILE: Cetak selama kondisi benar SET i = 1 WHILE i <= 5 DO OUTPUT i SET i = i + 1 ENDWHILE
D. Contoh Lengkap: Menghitung Luas Persegi Panjang
// Program: Hitung Luas Persegi Panjang // Input: panjang dan lebar // Output: luas START // Deklarasi variabel DECLARE panjang, lebar, luas AS NUMBER // Input dari pengguna OUTPUT "Masukkan panjang: " INPUT panjang OUTPUT "Masukkan lebar: " INPUT lebar // Proses perhitungan SET luas = panjang * lebar // Tampilkan hasil OUTPUT "Luas persegi panjang = " + luas END

Aturan Penulisan Pseudocode yang Baik

  1. Gunakan indentasi (spasi/tab) untuk menunjukkan hierarki
  2. Berikan komentar (diawali //) untuk penjelasan
  3. Gunakan kata kunci yang jelas: START, END, INPUT, OUTPUT, IF, THEN, ELSE, FOR, WHILE
  4. Setiap langkah harus spesifik dan tidak ambigu
  5. Gunakan huruf kapital untuk kata kunci (opsional, tapi disarankan)
Latihan Pseudocode!

Buatlah pseudocode untuk menghitung total harga pembelian dengan diskon 10% jika total > Rp100.000!

START INPUT total_harga IF total_harga > 100000 THEN SET diskon = total_harga * 0.10 SET total_bayar = total_harga - diskon OUTPUT "Diskon: Rp" + diskon ELSE SET total_bayar = total_harga OUTPUT "Tidak ada diskon" ENDIF OUTPUT "Total bayar: Rp" + total_bayar END

4. Model IPO (Input-Process-Output)

Apa itu Model IPO?

Model IPO adalah kerangka berpikir dasar dalam pemrosesan data yang terdiri dari tiga komponen: Input (masukan), Process (proses), dan Output (keluaran). Model ini adalah fondasi dari semua sistem komputer.

Diagram Model IPO
INPUT
Data yang dimasukkan ke sistem
PROCESS
Data diproses/diolah oleh sistem
OUTPUT
Hasil yang dihasilkan sistem

Penjelasan Detail Setiap Komponen

1. Input (Masukan)

Input adalah data atau informasi yang dimasukkan ke dalam sistem untuk diproses. Input bisa berasal dari:

  • Keyboard: Mengetik nama, angka, teks
  • Mouse: Klik, pilih menu, gambar
  • Scanner: Memindai dokumen/gambar
  • Sensor: Suhu, cahaya, gerakan
  • File: Data dari penyimpanan
2. Process (Proses)

Process adalah kegiatan mengolah input menjadi output. Proses bisa berupa:

  • Perhitungan: Menjumlahkan, mengurangi, mengalikan
  • Perbandingan: Membandingkan nilai (lebih besar, sama, dll)
  • Pengurutan: Mengurutkan data (A-Z, terbesar-kecil)
  • Pencarian: Mencari data tertentu
  • Transformasi: Mengubah format data
3. Output (Keluaran)

Output adalah hasil dari proses yang ditampilkan atau disimpan. Bentuk output:

  • Monitor: Tampilan visual (teks, gambar, video)
  • Printer: Cetakan dokumen
  • Speaker: Suara/audio
  • File: Data disimpan ke penyimpanan
  • Jaringan: Data dikirim ke perangkat lain

Contoh Penerapan Model IPO

KasusInputProcessOutput
KalkulatorDua angka, operator (+, -, *, /)Menghitung hasil operasiHasil perhitungan di layar
ATMKartu ATM, PIN, jumlah uangVerifikasi PIN, cek saldo, kurangi saldoUang tunai, struk transaksi
AbsensiSidik jari/NISN, waktuCocokkan data, catat waktuStatus hadir, laporan kehadiran
GameTekanan tombol, gerakan mouseHitung posisi, deteksi tabrakan, update skorGrafik game, skor, efek suara
Tips Mengidentifikasi IPO:
Tanyakan pada diri sendiri:
1. Input: "Apa yang saya masukkan ke sistem?"
2. Process: "Apa yang dilakukan sistem dengan data tersebut?"
3. Output: "Apa hasil yang saya dapatkan?"

5. Sistem Komputer Von Neumann

Sejarah Arsitektur Von Neumann

Arsitektur Von Neumann dikembangkan oleh matematikawan John von Neumann pada tahun 1945. Arsitektur ini menjadi dasar dari hampir semua komputer modern hingga saat ini.

Konsep Utama Von Neumann:
Program dan data disimpan dalam memori yang sama. Komputer membaca instruksi dari memori, mengeksekusinya, lalu melanjutkan ke instruksi berikutnya secara berurutan.

Komponen Arsitektur Von Neumann

Diagram Arsitektur Von Neumann
INPUT DEVICE (Keyboard, Mouse, Scanner)
CPU
ALU + Control Unit + Registers
MEMORY
RAM (Data & Program)
OUTPUT DEVICE (Monitor, Printer, Speaker)

Penjelasan Komponen

1. Central Processing Unit (CPU)

CPU adalah "otak" komputer yang menjalankan instruksi program. CPU terdiri dari:

  • ALU (Arithmetic Logic Unit): Melakukan operasi matematika (tambah, kurang, kali, bagi) dan logika (AND, OR, NOT)
  • Control Unit (CU): Mengatur dan mengontrol alur instruksi, mengambil data dari memori, mengirim hasil ke output
  • Registers: Memori kecil berkecepatan tinggi di dalam CPU untuk menyimpan data sementara
2. Memory (Memori)

Tempat menyimpan data dan program. Ada dua jenis utama:

RAM (Random Access Memory)
  • Memori sementara
  • Data hilang saat mati
  • Bisa dibaca dan ditulis
  • Contoh: DDR4, DDR5
ROM (Read Only Memory)
  • Memori permanen
  • Data tetap saat mati
  • Hanya bisa dibaca
  • Contoh: BIOS, Firmware
3. Input/Output Devices
Input Devices
  • Keyboard - memasukkan teks
  • Mouse - mengontrol kursor
  • Scanner - memindai gambar
  • Microphone - merekam suara
  • Webcam - merekam video
Output Devices
  • Monitor - menampilkan visual
  • Printer - mencetak dokumen
  • Speaker - mengeluarkan suara
  • Projector - proyeksi layar
  • Headphone - suara pribadi
4. Bus System (Jalur Komunikasi)

Bus adalah jalur komunikasi yang menghubungkan komponen-komponen komputer:

  • Data Bus: Mengirim data antar komponen
  • Address Bus: Menentukan lokasi memori yang diakses
  • Control Bus: Mengirim sinyal kontrol (baca/tulis)
Perbedaan Penting:
Hardware = Komponen fisik (bisa disentuh: CPU, RAM, monitor)
Software = Program/instruksi (tidak bisa disentuh: Windows, Word, Chrome)

6. State Machine dan Sistem Operasi

Apa itu State Machine?

State Machine (Mesin Keadaan) adalah model matematika yang menjelaskan perilaku sistem yang memiliki keadaan (state) tertentu dan dapat berpindah dari satu keadaan ke keadaan lain berdasarkan event (peristiwa) atau input.

Konsep Dasar State Machine:
State (Keadaan): Kondisi sistem pada suatu waktu
Transition (Transisi): Perpindahan dari satu state ke state lain
Event (Peristiwa): Pemicu yang menyebabkan transisi
Action (Aksi): Kegiatan yang dilakukan saat transisi

Contoh State Machine: Lampu Lalu Lintas

State Machine Lampu Lalu Lintas
MERAH
KUNING
HIJAU

Event: Timer (waktu habis) - Transisi ke state berikutnya

Contoh State Machine: Status Pesanan Online

// State Machine: Status Pesanan // States: DITERIMA - DIPROSES - DIKIRIM - DITERIMA_PELANGGAN STATE DITERIMA: ON event "bayar" - TRANSITION TO DIPROSES ON event "batal" - TRANSITION TO DIBATALKAN STATE DIPROSES: ON event "siap_kirim" - TRANSITION TO DIKIRIM STATE DIKIRIM: ON event "diterima" - TRANSITION TO SELESAI

Apa itu Sistem Operasi (OS)?

Sistem Operasi adalah perangkat lunak sistem yang mengelola sumber daya komputer (hardware) dan menyediakan layanan bagi program aplikasi. OS adalah penghubung antara pengguna dan hardware.

Fungsi Utama Sistem Operasi

Manajemen Proses

Mengatur jalannya program (proses), memberikan waktu CPU, dan menghentikan program yang bermasalah.

Manajemen Memori

Mengalokasikan dan mengatur penggunaan RAM agar program berjalan optimal.

Manajemen File

Mengatur penyimpanan, pengorganisasian, dan akses file di hard disk/SSD.

Manajemen Perangkat

Mengontrol perangkat keras (printer, scanner, USB) melalui driver.

2. Evaluasi Informasi Digital

Mengapa Perlu Mengevaluasi Informasi?

Internet dipenuhi informasi - ada yang benar, salah, menyesatkan, atau hoaks. Kemampuan mengevaluasi informasi sangat penting agar tidak terjebak misinformasi.

Bahaya Misinformasi:
Hoaks kesehatan - Risiko kesehatan | Berita palsu - Konflik sosial | Penipuan online - Kerugian finansial

5W+1H untuk Mengevaluasi Informasi

WHO

Siapa penulisnya? Apakah ahli di bidangnya?

WHAT

Apa isi informasinya? Fakta atau opini?

WHEN

Kapan dipublikasikan? Masih relevan?

WHERE

Di mana dipublikasikan? Situs kredibel?

WHY

Mengapa ditulis? Ada kepentingan apa?

HOW

Bagaimana datanya didapat? Ada bukti?

Ciri-Ciri Informasi Kredibel vs Hoaks

Informasi KredibelHoaks/Misleading
Penulis jelas dan bisa diverifikasiPenulis anonim atau palsu
Ada referensi/sumber dataTidak ada sumber atau sumber palsu
Bahasa netral dan objektifBahasa provokatif, emosional
Tanggal publikasi jelasTanggal tidak jelas atau palsu
Domain resmi (.go.id, .edu, .ac.id)Domain mencurigakan (.xyz, .tk)
Foto/video asli, tidak dieditFoto/video editan atau lama

Tools untuk Memeriksa Fakta

  • TurnBackHoax.id - Database hoaks Indonesia

  • MAFINDO - Masyarakat Anti Fitnah Indonesia

4. Konsep Jaringan Komputer

Apa itu Jaringan Komputer?

Jaringan komputer adalah kumpulan dua atau lebih komputer yang saling terhubung untuk berbagi sumber daya (data, printer, internet) dan berkomunikasi.

Jenis-Jenis Jaringan Berdasarkan Jangkauan

JenisSingkatanJangkauanContoh
PANPersonal Area NetworkBeberapa meterBluetooth, USB
LANLocal Area NetworkGedung/kampusWiFi sekolah, kantor
MANMetropolitan Area NetworkKotaJaringan antar kampus di satu kota
WANWide Area NetworkNegara/duniaInternet

Topologi Jaringan

Topologi adalah cara komputer disusun dan dihubungkan dalam jaringan.

Star (Bintang)

Semua komputer terhubung ke satu perangkat pusat (switch/hub). Jika satu kabel putus, hanya komputer tersebut yang terpengaruh.

Ring (Cincin)

Komputer terhubung membentuk lingkaran. Data mengalir searah. Jika satu kabel putus, seluruh jaringan terpengaruh.

Bus (Garis)

Semua komputer terhubung ke satu kabel utama (backbone). Jika kabel utama putus, seluruh jaringan mati.

Mesh (Jaring)

Setiap komputer terhubung ke semua komputer lain. Sangat andal tapi mahal dan kompleks.

Media Transmisi Jaringan

  • Kabel UTP (Unshielded Twisted Pair): Kabel tembaga yang paling umum digunakan. Ada kategori Cat5, Cat5e, Cat6, Cat6a.
  • Kabel Fiber Optic: Menggunakan cahaya untuk mengirim data. Kecepatan tinggi, tahan interferensi, jarak jauh.
  • Wireless (Nirkabel): Menggunakan gelombang radio (WiFi, Bluetooth, 4G/5G).
Tips:
Kecepatan internet diukur dalam Mbps (Megabit per second).
1 MB (Megabyte) = 8 Mb (Megabit). Jadi jika kecepatan 10 Mbps, download file 10 MB membutuhkan waktu sekitar 8 detik.

5. Keamanan Digital

Mengapa Keamanan Digital Penting?

Di era digital, data pribadi kita (foto, password, informasi bank) tersimpan di internet. Tanpa keamanan yang baik, data ini bisa dicuri, disalahgunakan, atau dihapus.

Ancaman Digital:
Phishing - Penipuan lewat email/pesan palsu | Malware - Virus, trojan, ransomware | Hacking - Peretasan akun | Identity Theft - Pencurian identitas

Password yang Kuat

Password adalah garis pertahanan pertama akun digital Anda. Password yang kuat harus:

  • Panjang: Minimal 12 karakter
  • Kompleks: Kombinasi huruf besar, huruf kecil, angka, dan simbol
  • Unik: Berbeda untuk setiap akun
  • Tidak mudah ditebak: Hindari tanggal lahir, nama, "password123"
Contoh Password Kuat:
B3l@j@rInformat1ka!2026 (menggabungkan kata, angka, simbol)

Contoh Password Lemah (JANGAN DIPAKAI!):
password123, 123456, nama123, tanggallahir

Autentikasi Dua Faktor (2FA)

2FA adalah lapisan keamanan tambahan selain password. Setelah memasukkan password, Anda harus memverifikasi identitas dengan cara kedua:

  • SMS/OTP: Kode dikirim ke nomor HP
  • Authenticator App: Google Authenticator, Microsoft Authenticator
  • Biometrik: Sidik jari, wajah, iris mata
  • Security Key: Perangkat fisik seperti USB token
Tips Keamanan Digital:
1. Aktifkan 2FA di semua akun penting (email, bank, media sosial)
2. Gunakan Password Manager (Bitwarden, LastPass) untuk menyimpan password
3. Jangan klik link mencurigakan di email/pesan
4. Update software dan antivirus secara rutin
5. Jangan gunakan WiFi publik untuk transaksi sensitif

Mengenali Phishing

Phishing adalah upaya penipuan untuk mendapatkan informasi sensitif (password, data bank) dengan menyamar sebagai entitas tepercaya.

Ciri-Ciri Email Phishing:
  • Alamat email pengirim mencurigakan (misal: support@amaz0n.com bukan amazon.com)
  • Teks berisi kesalahan ejaan/grammar
  • Minta informasi sensitif (bank, password)
  • Tautan yang mengarah ke situs palsu
  • Menciptakan rasa urgensi/kepanikan ("Akun Anda akan diblokir!")
  • Lampiran file mencurigakan (.exe, .zip dari pengirim tidak dikenal)

6. Hak Kekayaan Intelektual dan Etika Digital

Hak Kekayaan Intelektual (HKI)

Hak Kekayaan Intelektual adalah hak eksklusif yang diberikan kepada seseorang atas hasil karya intelektualnya. HKI melindungi kreativitas dan inovasi.

Jenis-Jenis HKI

JenisPerlindunganContohDurasi

Lisensi Software

Proprietary (Berbayar)

Software yang harus dibeli/license. Contoh: Microsoft Office, Adobe Photoshop, Windows. Dilarang menyalin/distribusikan tanpa izin.

Open Source (Gratis)

Kode sumber terbuka, bisa digunakan, dimodifikasi, dan didistribusikan. Contoh: Linux, LibreOffice, Firefox.

Pelanggaran HKI yang Sering Terjadi:
1. Mengunduh film/musik/software bajakan
2. Menyalin tugas/orang lain tanpa menyebut sumber (plagiarisme)
3. Menggunakan gambar dari internet tanpa izin
4. Memalsukan merek produk

Etika Digital

Etika digital adalah norma perilaku yang harus diterapkan saat menggunakan teknologi digital dan internet.

Prinsip Etika Digital:
Privasi

Menghargai privasi orang lain. Jangan membagikan informasi pribadi orang lain tanpa izin.

Berkomunikasi dengan Baik

Gunakan bahasa yang sopan. Hindari cyberbullying, ujaran kebencian, dan provokasi.

Kejujuran

Jangan menyebarkan hoaks, berita palsu, atau informasi menyesatkan.

Tanggung Jawab

Pikirkan dampak dari setiap unggahan. Apa yang diinternet, tetap diinternet.

Penyaringan Konten dan Pengelolaan Akun

Tips Mengelola Akun Digital dengan Bijak:
  1. Gunakan email yang berbeda untuk akun penting (bank) dan akun umum (media sosial)
  2. Review privasi di setiap platform sosial media secara berkala
  3. Jangan oversharing - batasi informasi pribadi yang dibagikan
  4. Gunakan fitur penyaringan (filter) untuk konten yang tidak pantas
  5. Laporkan konten negatif, penipuan, atau cyberbullying
  6. Hapus akun lama yang tidak lagi digunakan
  7. Aktifkan parental control jika diperlukan
Digital Footprint (Jejak Digital):
Setiap aktivitas online meninggalkan jejak. Foto, komentar, unggahan yang dihapus masih bisa tersimpan di server atau di-screenshot orang lain. Berpikirlah sebelum memposting!
